The Trigger widget¶

Author: Christian Mayer
Available since version: 2012

Description¶

Adds a button to the visu with which exactly a defined value for a short, as well as a defined value for a long key pressure, can be sent to the BUS, e.g. for recalling and storing scenes or driving roller blinds. (Short = stop, long = drive). The address for short and long term may vary.

Settings¶

For a general understanding of how the configuration files are structured and what elements and attributes are it is recommended to read this section first: Pages and structure of CometVisu.

The behaviour and appearance of the Trigger widget can be influenced by using certain attributes and elements. The following tables show the allowed attributes and elements and their possible values. The screenshots show, how both can be edited in the editor.

Attributes underlined by ….. are mandatory, all the others are optional and be omitted.

Allowed attributes in the Trigger-element¶

Element Attribute
  Name Content Description
trigger value string  
shortvalue string Defines the value which is sent on a short press of the button
shorttime decimal Time in milliseconds that is accepted as shorttime.
mapping string Map the bus value to a different value, text or symbol for displaying. See also Mapping
styling string Change the color of the displayed value depending on its value. See also Styling
align left, right or center  
flavour string Selection of a display variant. See also Flavour.
bind_click_to_widget true or false use the whole widget area to react on click events
class string Add this value to the CSS class so that it can be formatted by a user provided style sheet.
send-long-on-release true or false Fire the long press event after the button is released (=true) or immediately after the threshold is exceeded (=false).
format string Formatting of the value (printf syntax).

Examples¶

Make sure that you only use UTF-8 encoded characters by settings the encoding in your XML-editor to UTF-8 mode!

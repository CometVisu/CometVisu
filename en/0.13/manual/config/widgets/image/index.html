The Image widget¶

Author: Christian Mayer
Available since version: 0.8.0 (2012)

Description¶

Adds an image to your visualization. Using the auto-refresh setting this widget can be used e.g. to show a camera picture.

Settings¶

For a general understanding of how the configuration files are structured and what elements and attributes are it is recommended to read this section first: Pages and structure of CometVisu.

The behaviour and appearance of the Image widget can be influenced by using certain attributes and elements. The following tables show the allowed attributes and elements and their possible values. The screenshots show, how both can be edited in the editor.

Attributes underlined by ….. are mandatory, all the others are optional and be omitted.

Allowed attributes in the Image-element¶

Element Attribute
  Name Content Description
image src URI Complete path (URI) to the picture
width dimension (number+unit) Sets the width of the image. Format must be CSS (e.g. px, em, %).
height dimension (number+unit) Sets the height of the image. Format must be CSS (e.g. px, em, %).
widthfit true or false  
crop-top dimension (number+unit) Sets the top crop of the image. Format must be absolute CSS (e.g. px, em).
crop-bottom dimension (number+unit) Sets the bottom crop of the image. Format must be absolute CSS (e.g. px, em).
refresh decimal Sets the update rate for the picture in seconds
cachecontrol full, force, weak or none Set the cache control strategy for refresh.
flavour string Selection of a display variant. See also Flavour.
class string Add this value to the CSS class so that it can be formatted by a user provided style sheet.
align left, right or center  
placeholder none, src, hide or exclude Defined how to deal with the case when no image url is set

Valid values for cachecontrol are:

full Standard. By extending the URL with a timestamp a refresh is
ensured. Server support is required though.
force Using sophisticated methods an enforcement of the refresh is
attempted.
weak The URL will be extended by an anchor with timestamp. This will
work with most servers where full mode doesn’t work. But it requires the correct use of HTTP headers by the server as well as a correct reaction by the web browser.
none The URL will not be modified. A working refresh can be prevented
by caching.

Elements in the editor¶

Examples¶

It is possible to manually edit the visu_config.xml and add an entry for the Image widget.

Caution

Make sure that you only use UTF-8 encoded characters by settings the encoding in your XML-editor to UTF-8 mode!

<image src="resource/icons/CometVisu_orange.png" width="45px" height="32px">
  <layout colspan="2"/>
</image>
